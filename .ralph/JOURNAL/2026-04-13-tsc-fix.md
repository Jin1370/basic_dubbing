- [P0] tsc --noEmit 실행하여 전체 타입 에러 수정

## 접근 방식
1. `npx tsc --noEmit` 실행 → 에러 1건: `expo-auth-session` 모듈 누락
2. `npm install expo-auth-session --legacy-peer-deps`로 패키지 설치
   - `npx expo install`은 react-dom peer dependency conflict로 실패
   - `--legacy-peer-deps` 사용 결정 (expo-auth-session은 react-dom에 의존하지 않으므로 안전)
3. 재실행 → 에러 1건: `useProxy` 옵션이 `AuthSessionRedirectUriOptions`에 존재하지 않음
   - expo-auth-session 최신 버전에서 `useProxy` 옵션 제거됨
   - `AuthSession.makeRedirectUri()` 호출에서 `{ useProxy: true }` 인자 제거
4. 재실행 → 에러 0건, 통과

## 변경 파일
- `lib/auth.ts` — `makeRedirectUri({ useProxy: true })` → `makeRedirectUri()` (line 20)
- `package.json` / `package-lock.json` — `expo-auth-session` 의존성 추가
