## 접근 방식

### 401 자동 로그아웃
- `lib/api.ts` 응답 인터셉터에서 401 수신 시 `signOut()` 호출 추가
- `signOut()`은 fire-and-forget (`catch(() => {})`) — signOut 실패가 사용자 에러 흐름을 막지 않도록
- signOut이 Supabase auth state를 변경하면, store의 `onAuthStateChange` 리스너가 세션을 null로 설정 → `_layout.tsx`가 auth 그룹으로 리다이렉트

### SafeAreaProvider 래핑
- `app/_layout.tsx`의 로딩 상태와 메인 `<Slot />` 모두 `<SafeAreaProvider>`로 래핑
- `react-native-safe-area-context`는 이미 package.json에 존재

## 변경 파일
- `lib/api.ts` — signOut import 추가, 401 핸들러에 signOut() 호출 추가
- `app/_layout.tsx` — SafeAreaProvider import 및 래핑 추가
